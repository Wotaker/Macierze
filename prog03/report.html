Pseudokod algorytmu rekurencyjnej LU faktoryzacji¶

def lu_decomposition(A: Matrix) -> Matrix, Matrix:

    n = size(A)     # rozmiar macierzy A

    # Warunek brzegowy - macierz L skada si z jedynki, 
    # a macierz U skada si z oryginalnej wartoci na przektnej A
    if n == 1:
        return [[1]], A

    # Podzia macierzy A na bloki o jednakowym rozmiarze
    [A11 A12]
    [A21 A22] = A

    # Obliczamy LU faktoryzacj dla lewej grnej macierzy
    L11, U11 = lu_decomposition(A11)

    # Obliczamy lew dolncz LU faktoryzacji
    U11_inv = inverse(U11)
    L21 = A21 * U11_inv
    U21 = zeros(m, m)  # grna trjktna macierz w lewej dolnej czci skada si z zer

    # Obliczamy praw grn cz LU faktoryzacji
    L11_inv = inverse(L11)
    U12 = L11_inv * A12
    L12 = zeros(m, m)  # dolna trjktna macierz w prawej grnej czci skada si z zer

    # Obliczamy praw doln (najtrudniejsz) cz LU faktoryzacji
    S = A22 - (A21 * U11_inv) * (L11_inv * A12)
    L22, U22 = lu_decomposition(S)  # obliczamy rekurencyjnie

    # Skadanie obliczonych blokw w wynikowe macierze L i U
    L = [L11, L12]
        [L21, L22]

    U = [U11, U12]
        [U21, U22]

    return L, U

Pseudokod rekurencyjnego algorytmu odwracania macierzy¶

def inverse(A: Matrix) -> Matrix:

    n = size(A)     # rozmiar macierzy A

    # Warunek brzegowy - trywialna odwrotno
    if n == 1:
        return 1 / A

    # Podzia macierzy A na bloki o jednakowym rozmiarze
    [A11 A12]
    [A21 A22] = A

    # Zmienne pomocnicze
    A11_inv = inverse(A11)
    S22 = A22 - A21 * A11_inv * A12
    S22_inv = inverse(S22)

    # Oblicznie blokw skadajcych si na macierz wynikow B
    B11 = A11_inv + A11_inv * A12 * S22_inv * A21 * A11_inv
    B12 = -A11_inv * A12 * S22_inv
    B21 = -S22_inv * A21 * A11_inv
    B22 = S22_inv

    # Skadanie obliczonych blokw w wynikow macierz B
    B = [B11, B12]
        [B21, B22]

    return B

Pseudokod algorytmu rekurencyjnego mnożenia macierzy¶

def multiply_strassen_with_classic(A: Matrix, B: Matrix, size_classic: int = 8) -> Matrix:

    def strassen(A: Matrix, B: Matrix, n: int):

        # Warunek brzegowy - trywialne mnoenie
        if n == 1:
            return A * B

        # Warunek brzegowy - rozmiar macierzy suboptymalny aby wywoywa rekurencyjnie
        elif n <= size_classic:
            return multiply_classic(A, B)

        # Podzia macierzy A na bloki o jednakowym rozmiarze
        [A11 A12]
        [A21 A22] = A

        # Podzia macierzy B na bloki o jednakowym rozmiarze
        [B11 B12]
        [B21 B22] = B

        # Wywoania rekurencyjne aby wyliczy macierze pomocnicze P_i, i z {1, 2, ..., 7}
        P1 = strassen(A11 + A22, B11 + B22, n // 2)
        P2 = strassen(A21 + A22, B11, n // 2)
        P3 = strassen(A11, B12 - B22, n // 2)
        P4 = strassen(A22, B21 - B11, n // 2)
        P5 = strassen(A11 + A12, B22, n // 2)
        P6 = strassen(A21 - A11, B11 + B12, n // 2)
        P7 = strassen(A12 - A22, B21 + B22, n // 2)

        # Skadanie macierzy wynikowej C z blokw powstaych z P_i
        C11 = P1 + P4 - P5 + P7
        C12 = P3 + P5
        C21 = P2 + P4
        C22 = P1 - P2 + P3 + P6

        C = [C11, C12]
            [C21, C22]

        return C

    return strassen(A, B, size(A))

FLO: liczba operacji zmiennoprzecinkowych w algorytmie¶

Niech:
$lu(n)$ - liczba operacji zmiennoprzecinkowych (FLO) dla rekurencyjnej LU faktoryzacji dla macierzy rozmiaru $n$, $inv(n)$ - liczba operacji zmiennoprzecinkowych (FLO) dla rekurencyjnego odwracania macierzy rozmiaru $n$,
$str(n)$ - FLO dla rekurencyjnego mnozenia macierzy algorytmem Strassen'a,
$sum(n)$ - FLO dla sumowania macierzy rozmiaru $n$.

$lu(2^0) = 1$

$lu(2^k) = lu(2^{k-1}) + 2inv(2^{k-1}) + 2str(2^{k-1}) + 3str(2^{k-1}) + sum(2^{k-1}) + lu(2^{k-1})$

$= 2lu(2^{k-1}) + 2inv(2^{k-1}) + 5str(2^{k-1}) + sum(2^{k-1})$

Niech $q(k) = 2inv(2^{k}) + 5str(2^{k}) + sum(2^{k})$,

Wtedy mamy:

$lu(2^k) = 2lu(2^{k-1}) + q(k-1)$

$= 2\Big(2lu(2^{k-2}) + q(k-2)\Big) + q(k-1)$

$= 2\Bigg(2\Big(2lu(2^{k-3}) + q(k-3)\Big) + q(k-2)\Bigg) + q(k-1)$

$= \dots $

$= 2^{k} + \sum_{i=1}^{k} 2^{k-i}q(i-1)$

Jeli rozpiszemy $q(k)$ - to dostaniemy:

$q(k) = 2 \cdot \Bigg(17 \cdot 2^{k-1} + \sum^{k}_{i=2} 2^{k-i}r(i-1)\Bigg) + 5 \cdot \Bigg(\frac{23}{5} \cdot 7^k - \frac{78}{5} \cdot 2^k\Bigg) + 4^k$

Eksperymenty¶

Eksperymenty polegaj zbadaniu zoonoci algorytmu rekurencyjnego odwracania macierzy poprzez:

  • Pomiar zoonoci czasowej (rednia artmetyczna z 15 pomiarw)
  • Obliczenie liczby wykonanych operacji zmiennoprzecinkowych

Wyniki zebralimy dla macierzy rozmiaru $2^k\times2^k, k \in \{1, 2, ..., 7\}$. Ograniczenie na rozmiar wyniko z czasu wykonania algorytmu dla duych macierzy ($k > 7$), poniewa by on ju liczony w minutach i nie byo moliwe w sensownym czasie wyliczy redniej z czasu wykonania. Warto progowa dla mnozenia z uzyciem algorytmu Strassena zostaa dobrana empirycznie i wynosi $l=3$.

Wykres grny ma zarwno o rozmiaru, jak i czasu i zoonoci w skali logarytmicznej, przez co zoono wykadnicza prezentuje si w tej reprezentacji liniowo. Proste aproksymujce odpowiednio zlozonosc czasow LU faktoryzacji maych ($n <= 4$) i duych ($n >= 8$) macierzy, s nachylone pod rnymi ktami. Pokazuje to, ze faktycznie w algorytmie wystpuje pewna wartoc progowa warunkujca jego dziaanie i jest to prg $l = 3 \rightarrow n = 8$, ktry odpowiada za dobr algorytmu mnozenia macierzy (rekurencyjny Strassena lub klasyczny). Doskonale to wida na pierwszym rysunku zaczynajc od macierzy $8x8$, czyli naszego progu. Im dalej idziemy (dla wikszych macierzy) efekty uboczne trac wpyw i warto praktyczna zblia si do teoretycznej.

Z kolei dolny wykres na ktrym tylko o rozmiaru jest w skali logarytmicznej, ukazuje wykadniczy charakter zozonoci algorytmu. Widac na nim, e zoonoci czasowa oraz liczby operacji s silnie skolerowane, co jest zgodne z intuicj.
